# context.ts (API) — Brief + Junior Q&A

**Path:** `apps/api/src/context.ts`  
**Role:** Build GraphQL `context` object from Express `req` for every resolver.

---

## Junior Q&A

--------------------------------------------------------------------------------------------------------------------------------------------
**[0] What is GraphQL context?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Third argument to resolvers — holds `user`, `tenantId`, `req`, etc. — interface `context.ts:7-19`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[1] How is tenant subdomain resolved?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `req.subdomain` → `req.tenant.subdomain` → `x-tenant` header → `'demo'` — `context.ts:22`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[2] Where does `user` come from?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** JWT middleware sets `req.user` before GraphQL runs — copied in `context.ts:27`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[3] `tenant` vs `tenantId` — which for DB queries?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Always **`tenantId`** (Mongo id). `tenant` is subdomain string only — `context.ts:11-16`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[4] When is `buildGraphQLContext` used instead?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** WebSocket subscriptions / tests — no full Express stack. See `context/buildContext.ts`.
