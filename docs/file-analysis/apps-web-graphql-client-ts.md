# client.ts (Apollo) — Brief + Junior Q&A

**Path:** `apps/web/graphql/client.ts`  
**Role:** Single Apollo Client — HTTP + WebSocket, auth headers, 401 handling.

---

## Junior Q&A

--------------------------------------------------------------------------------------------------------------------------------------------
**[0] What is the link chain order?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `errorLink` → `authLink` → `httpLink` or WebSocket — `client.ts:103`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[1] How is the Bearer token attached?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `authLink` reads `localStorage` `authToken`, sets `authorization` header — `client.ts:55-77`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[2] Why is `x-tenant` from the host, not localStorage?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Avoid stale tenant when user switches subdomain — `resolveRequestTenant()` — `client.ts:75`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[3] When does the client clear session and redirect?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Expired token or tenant mismatch in `authLink` (`60-68`); API 401 in `errorLink` (`95-99`). Skips Login/Register ops (`87-90`).

--------------------------------------------------------------------------------------------------------------------------------------------
**[4] How do subscriptions connect?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `splitLink` routes subscription operations to `graphql-ws`; `connectionParams` re-reads token on reconnect — `client.ts:20-47`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[5] What is `errorPolicy: 'all'`?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Return partial data even if some GraphQL fields error — `client.ts:105-108`.
