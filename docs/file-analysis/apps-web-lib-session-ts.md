# session.ts — Brief + Junior Q&A

**Path:** `apps/web/lib/session.ts`  
**Role:** Save JWT + user in browser after login; clear on logout/401.

---

## Junior Q&A

--------------------------------------------------------------------------------------------------------------------------------------------
**[0] What does this file do?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Stores login token and user JSON in `localStorage`, mirrors cookies for SSR, dispatches `luxgen-auth-change` on login/logout.

--------------------------------------------------------------------------------------------------------------------------------------------
**[1] What keys are used in localStorage?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `authToken`, `currentUser`, `authTokenExpiresAt`, `authSessionEpoch` — see `session.ts:27-34`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[2] What is `SessionUser`?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Type for logged-in user: id, email, name, role, `tenant.id`, `tenant.subdomain` — `session.ts:6-18`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[3] How does the client check if the token expired?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Read JWT `exp` from payload (`decodeJwtPayload` `74-83`), compare to now with 30s skew — `isTokenExpired` `91-96`, `isStoredSessionExpired` `106-115`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[4] What runs on login success?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `persistSession(token, user)` — writes storage, cookies, bumps epoch, fires event — starts at `118`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[5] What runs on logout?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `clearStoredSession()` — removes all auth keys + cookies — starts at `172`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[6] Can we trust `luxgen_user` in localStorage alone?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** **No.** Display cache only. Need valid `authToken` + non-expired session.

--------------------------------------------------------------------------------------------------------------------------------------------
**[7] Who listens to `luxgen-auth-change`?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `AuthGuard.tsx:39`, `app-layout-header.ts:16`, layout user provider — re-check session without full reload.
