# Junior Q&A — Node, MongoDB, GraphQL, Auth (LuxGen)

> Short answers + **file:line** pointers.

____________________________________________________________________________________________________________________________________________

:::::::::::::::::::::::::::::::::::::::::::::::::::::::::::: NODE & EXPRESS ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
____________________________________________________________________________________________________________________________________________

--------------------------------------------------------------------------------------------------------------------------------------------
**[0] What is Node.js?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** JavaScript runtime on the server (V8). LuxGen API runs on Node — `apps/api/src/index.ts` starts Express.

--------------------------------------------------------------------------------------------------------------------------------------------
**[1] What is Express middleware?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Functions `(req, res, next)` that run in order. Each can read/write `req`, end response, or call `next()`.

**In LuxGen:** `apps/api/src/app.ts:43-50` — `helmet`, `cors` run before routes.

--------------------------------------------------------------------------------------------------------------------------------------------
**[2] Why does middleware order matter?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Later middleware sees what earlier middleware attached to `req`.

**In LuxGen order (`app.ts`):**
1. Security (`helmet`, `cors`) — ~43-50
2. `tenantRoutingMiddleware` — resolves tenant from subdomain
3. `authMiddleware` — JWT → `req.user`
4. GraphQL handler — reads `context()` from `req`

--------------------------------------------------------------------------------------------------------------------------------------------
**[3] What is REST vs GraphQL in LuxGen?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.**
- **GraphQL** — main app API (`/graphql`) — courses, users, billing.
- **REST** — webhooks, auth helpers, Stripe — `apps/api/src/routes/`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[4] What is JWT?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Signed token: `header.payload.signature`. Server verifies signature; payload holds `userId`, `tenant`, `exp`.

**Client:** stores in `localStorage` key `authToken` — `apps/web/lib/session.ts:27-28`

**Server:** `authMiddleware` validates and sets `req.user`.

:::::::::::::::::::::::::::::::::::::::::::::::::::::::::::: GRAPHQL ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
____________________________________________________________________________________________________________________________________________

--------------------------------------------------------------------------------------------------------------------------------------------
**[5] What is GraphQL context?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Object passed to every resolver: `user`, `tenantId`, `req`, etc.

**In LuxGen:** `apps/api/src/context.ts:7-19` — interface; `context.ts:21-34` — built from Express `req`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[6] Difference between `tenant` and `tenantId` in context?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.**
- `tenant` — **subdomain string** (`demo`) — legacy resolver field
- `tenantId` — **Mongo ObjectId string** — use for DB queries

**Read:** `apps/api/src/context.ts:11-16`

--------------------------------------------------------------------------------------------------------------------------------------------
**[7] How does the web app send auth to GraphQL?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Apollo `authLink` adds headers on every request:

- `Authorization: Bearer <token>`
- `x-tenant: <subdomain from host>`

**Read:** `apps/web/graphql/client.ts:55-77`

--------------------------------------------------------------------------------------------------------------------------------------------
**[8] What happens on GraphQL 401?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `errorLink` clears session and redirects to login (except Login/Register mutations).

**Read:** `apps/web/graphql/client.ts:80-100`

--------------------------------------------------------------------------------------------------------------------------------------------
**[9] What is a GraphQL subscription?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Long-lived WebSocket connection for live updates.

**In LuxGen:** `client.ts:38-47` — `split` sends subscriptions to `graphql-ws`, queries to HTTP.

:::::::::::::::::::::::::::::::::::::::::::::::::::::::::::: MONGODB :::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
____________________________________________________________________________________________________________________________________________

--------------------------------------------------------------------------------------------------------------------------------------------
**[10] What is MongoDB?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Document database — JSON-like **documents** in **collections**. LuxGen uses **Mongoose** ODM in `packages/db/`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[11] What is a Mongoose schema?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Defines fields, types, validation, indexes for a collection.

**In LuxGen:** `packages/db/src/tenant.ts` — `tenantSchema` with `subdomain`, `status`, embedded `settings`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[12] What is multi-tenancy in LuxGen?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Many customers (tenants) share one app. Data isolated by `tenantId` on documents. URL subdomain picks tenant (`demo.localhost`).

**Read:** `packages/db/src/tenant.ts:107-114` — unique `subdomain` field.

--------------------------------------------------------------------------------------------------------------------------------------------
**[13] What is an index?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Speeds up queries (like a book index). Without index, Mongo scans every document.

**In LuxGen:** `packages/db/src/tenant.ts:292-294` — indexes on `subdomain`, `domain`, `status`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[14] What is `OverwriteModelError`?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Mongoose model registered twice (hot reload or duplicate `.js` + `.ts`).

**Fix in LuxGen:** `packages/db/src/tenant.ts:295-296` — reuse `mongoose.models.Tenant` if exists.

--------------------------------------------------------------------------------------------------------------------------------------------
**[15] Embed vs reference?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.**
- **Embed** — nested object inside document (tenant branding in `settings`)
- **Reference** — store `ObjectId`, `populate()` later (user → tenant)

**In LuxGen:** Tenant embeds branding; User document has `tenantId` reference.

:::::::::::::::::::::::::::::::::::::::::::::::::::::::::::: AUTH (LUXGEN) ::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::::
____________________________________________________________________________________________________________________________________________

--------------------------------------------------------------------------------------------------------------------------------------------
**[16] Where is the session stored in the browser?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `localStorage`:

| Key | Line |
|-----|------|
| `authToken` | `session.ts:28` |
| `currentUser` | `session.ts:29` |
| `authTokenExpiresAt` | `session.ts:31` |

**Read:** `apps/web/lib/session.ts:27-34`

--------------------------------------------------------------------------------------------------------------------------------------------
**[17] What runs after successful login?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `persistSession(token, user)` writes localStorage + cookies + fires `luxgen-auth-change`.

**Read:** `apps/web/lib/session.ts:118` (function start)

--------------------------------------------------------------------------------------------------------------------------------------------
**[18] How does the app check token expiry on the client?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Decode JWT `exp` claim (no signature verify on client). 30s skew buffer.

**Read:** `apps/web/lib/session.ts:91-96` — `isTokenExpired`; `106-115` — `isStoredSessionExpired`

--------------------------------------------------------------------------------------------------------------------------------------------
**[19] What is tenant mismatch?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** User logged in on `demo` but opened `idea-vibes.localhost` — session cleared, redirect `tenant_mismatch`.

**Read:** `apps/web/lib/tenant-auth.ts:19-24` — `isSessionTenantMismatch`

--------------------------------------------------------------------------------------------------------------------------------------------
**[20] Never compare JWT tenant to subdomain string — why?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** JWT `tenant` claim is **Mongo ObjectId**. Subdomain is `demo`. Comparing them is always wrong.

**Use:** `user.tenant.subdomain` vs host subdomain (`tenant-auth.ts`).

--------------------------------------------------------------------------------------------------------------------------------------------
**[21] How is sidebar menu filtered by role?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** `useSidebarSections` reads role + billing flags, filters default nav items.

**Read:**
- Guest allowlist — `use-sidebar-sections.ts:33-35`
- Admin-only items — `use-sidebar-sections.ts:38-42`
- Hook entry — `use-sidebar-sections.ts:85-104`

--------------------------------------------------------------------------------------------------------------------------------------------
**[22] Is sidebar filtering enough for security?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** **No.** UI only hides links. API resolvers must check `context.user` and `context.tenantId`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[23] What ports run locally?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Web **3000**, API **4000**, Mongo **27017**, Redis **6379** — see `Makefile` / `docs/DEVELOPER_GUIDE.md`.

--------------------------------------------------------------------------------------------------------------------------------------------
**[24] What is `make dev-stack-admin`?**
--------------------------------------------------------------------------------------------------------------------------------------------

**ANS.** Starts Mongo, Redis, API, web for admin/dev workflow. Run after pulling main.
